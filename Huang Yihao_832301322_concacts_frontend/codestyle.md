# 代码风格规范 (codestyle.md)

## 1. 通用原则
- 代码需保持清晰可读，优先考虑可读性而非简洁性
- 所有文件使用 UTF-8 编码
- 缩进统一使用 4 个空格（不使用制表符）
- 每行代码长度控制在 80 字符以内，过长需换行
- 代码块之间保留一个空行分隔，增强可读性
- 禁止出现无意义的注释（如重复代码功能的注释）

## 2. HTML 规范

### 2.1 文档结构
- 所有 HTML 文件需包含完整的 DOCTYPE 声明：`<!DOCTYPE html>`
- 统一使用语义化标签（`<header>`, `<main>`, `<section>`, `<footer>` 等），避免过度使用 `<div>`
- 每个页面结构需包含：`<head>`（含元信息）和 `<body>`（含内容）
- `<meta charset="UTF-8">` 必须放在 `<head>` 最前面，确保字符编码正确

### 2.2 标签规范
- 标签名统一使用小写字母（如 `<div>` 而非 `<DIV>`）
- 自闭合标签需明确闭合（如 `<img/>`, `<input/>`）
- 标签属性值必须用双引号包裹（如 `class="container"` 而非 `class='container'`）
- 避免冗余属性（如 `<form method="get">` 无需重复默认 method）
- 表单元素必须添加 `name` 或 `id` 属性，且值需唯一

### 2.3 注释规范
- HTML 注释格式：`<!-- 注释内容 -->`
- 注释需说明代码块的功能（如 `<!-- 联系人列表区域 -->`），而非描述标签本身
- 重要逻辑或复杂结构需添加注释，简单结构可省略

## 3. CSS 规范

### 3.1 样式文件
- 所有样式统一写在 `style.css` 中，禁止使用内联样式（`style` 属性）
- 引入外部样式时使用 `<link>` 标签（放在 `<head>` 中），禁止使用 `@import`

### 3.2 选择器规范
- 类名使用小写字母，多个单词用短横线连接（如 `.contact-card` 而非 `.ContactCard` 或 `.contact_card`）
- ID 名唯一且具有语义（如 `#add-form` 而非 `#form1`）
- 避免使用标签选择器（如 `div{}`），优先使用类选择器
- 减少选择器嵌套层级（最多不超过 3 层）
