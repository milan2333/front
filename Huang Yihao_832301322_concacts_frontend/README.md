# 联系人管理系统 - 前端

## 项目概述
这是联系人管理系统的前端应用，提供直观的界面用于管理联系人信息。用户可通过该应用实现联系人的查看、添加、编辑、删除、搜索和排序等操作，操作流程简洁高效。

## 功能特点
- **联系人列表展示**：以卡片或列表形式展示所有联系人，包含姓名、电话、分类等核心信息
- **新增联系人**：通过带验证的表单添加新联系人，确保信息完整性
- **编辑联系人**：点击编辑按钮可修改现有联系人信息，自动回显当前数据
- **删除联系人**：删除操作需二次确认，防止误删重要联系人
- **搜索筛选**：支持按姓名、电话等关键词快速定位联系人
- **响应式布局**：适配电脑、平板和手机等不同设备屏幕
- **操作反馈**：添加、修改、删除后显示相应提示信息，提升用户体验

## 技术栈
- HTML5：构建语义化页面结构
- CSS3：负责页面样式美化，包括布局、颜色和动画效果
- JavaScript（ES6+）：实现表单验证、数据交互和页面动态效果
- 外部资源：使用Font Awesome图标增强视觉表现

## 快速上手
### 前置要求
- 现代浏览器（推荐Chrome、Firefox、Edge最新版本）
- 已部署并运行的后端服务（默认对接地址：http://localhost:7894）

### 部署步骤
1. 将前端文件（home.html、index.html、add.html、edit.html、style.css）复制到同一目录
2. 通过浏览器直接打开index.html或home.html即可使用
3. 若后端服务地址不同，需修改各页面中JavaScript的请求地址

## 文件结构
```markdown
frontend/
├── home.html       # 首页，展示联系人列表及操作入口
├── index.html      # 系统入口页面
├── add.html        # 添加联系人表单页面
├── edit.html       # 编辑联系人表单页面
├── style.css       # 全局样式文件，统一页面风格
└── README.md       # 项目说明文档
```


## 浏览器兼容性
- Chrome 80+
- Firefox 75+
- Edge 80+
- Safari 13+

## 常见问题
- **样式异常**：检查 style.css 路径是否正确
- **无法提交表单**：确认必填项已填写，后端服务已启动
- **数据不更新**：刷新页面或检查后端接口状态

## 注意事项
- 需配合后端服务使用，单独运行无法持久化数据
- 生产环境建议前后端部署在同一域名，避免跨域
